# Frontend Design

## Overview
Design and build frontend UI — components, pages, layouts.

## When to trigger
- User wants to create or improve web UI
- Tasks involving HTML, CSS, JS, React, Vue, Svelte, or Tailwind

## Principles
- Mobile-first responsive design
- Accessible by default (ARIA, keyboard nav, contrast)
- Semantic HTML over div soup
- CSS custom properties for theming

## Workflow
1. Clarify framework, design system, and target devices
2. Build component or page
3. Test in browser (if possible) before reporting done

## Anti-patterns to avoid
- Inline styles for anything reusable
- Hardcoded pixel values instead of relative units
- Missing alt text on images
