# Internal Comms

## Overview
Draft clear, professional internal communications.

## When to trigger
- User needs to write an announcement, memo, team update, or internal email
- Meeting notes, retrospective summaries, or all-hands content

## Formats supported
- Email (subject + body)
- Slack/Teams message (concise, emoji optional)
- Memo (header + sections)
- Meeting notes (agenda → decisions → action items)

## Principles
- Lead with the key message — don't bury the lede
- Action items: owner + deadline always explicit
- Match the company's existing tone (ask if unknown)
- Shorter is better for async comms
